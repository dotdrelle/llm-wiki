# Articulation des notes projet

## Notes attendues

1. Note de cadrage
2. Registre des exigences
3. Options d’architecture
4. Comparaison et décision
5. Exploitation
6. Journal et lacunes

## Règle de responsabilité

Chaque note a une responsabilité unique.

- Le cadrage décrit le besoin et le périmètre.
- Le registre des exigences maintient les IDs d’exigences.
- Les options d’architecture décrivent les scénarios candidats.
- La comparaison et décision compare les options et trace les arbitrages.
- L’exploitation décrit les impacts opérationnels.
- Le journal et lacunes trace l’évolution, les points ouverts et les manques.

## Règle anti-dérive

Les exigences ne sont créées que dans le registre des exigences.
Les options ne sont décrites complètement que dans la note d’options d’architecture.
Les décisions ne sont consignées que dans la note de comparaison et décision.
Les lacunes sont centralisées dans la note journal et lacunes.

Les autres notes peuvent référencer ces éléments mais ne doivent pas les recréer.

## Règle de mise à jour

Une note doit représenter l’état courant documenté.

Si les sources sont contradictoires, la note doit le signaler au lieu d’arbitrer sans preuve.

Si les sources sont incomplètes, la note doit créer une lacune.
