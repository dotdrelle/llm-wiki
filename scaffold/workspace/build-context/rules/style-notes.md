# Style des notes projet

Les notes projet doivent être courtes, lisibles et maintenables.

Privilégier :

- l’état courant ;
- les tableaux ;
- les IDs stables ;
- les points ouverts ;
- les liens entre exigences, options et décisions.

Éviter :

- les longs développements ;
- les répétitions ;
- les conclusions non sourcées ;
- les sections vides sans mention de lacune.
